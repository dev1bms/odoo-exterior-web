{# ==================================================================== PROBLEM ==================================================================== #}
The problem

Studio is fast. Tracking what it changed isn't.

Odoo Studio lets anyone customize a database in clicks — fields, views, automations, menus, security. After a few months, that knowledge becomes scattered across screens and admins.

Customizations get scattered

Fields, views, server actions and rules end up across dozens of technical menus, with little overview.

Hard to hand over

New consultants or developers struggle to map what was changed and why before touching anything.

Risky to migrate

Rebuilding Studio work as a custom module without a clean inventory is where regressions sneak in.

{# ==================================================================== SOLUTION ==================================================================== #}
The solution

One read-only scan, one structured report.

Odoo Exterior queries your Odoo instance over standard XML-RPC, using a strict allowlist of read-only methods, and produces a single artifact you can review, version, and hand to a developer.

{# ==================================================================== HOW IT WORKS ==================================================================== #}
How it works

From credentials to a structured report in four steps.

1
Connect your Odoo

Add an instance with URL, database, username, and a password or API key. Stored encrypted.

2
Test read-only access

Verify authentication before doing anything. Only read methods are ever called.

3
Run the Studio audit

Models, fields, views, actions, menus and security are scanned in a single pass.

4
Download the reports

Get studio_report.md for humans and studio_data.json for tooling.

{# ==================================================================== FEATURES ==================================================================== #}
Features

Everything you need to map a Studio database.

A focused toolset, not a Swiss-army knife. Each module is read-only, opinionated, and produces auditable output.

Studio field extraction

Detects x_studio_* and manual fields, exports type, relation, required, readonly, stored, computed code, and dependencies.

Custom model discovery

Lists Studio-generated models and any standard model that has been extended with Studio fields.

View reverse engineering

Captures modified and Studio-generated views with their inheritance chain and full arch_db.

Server action review

Server actions and base.automation rules with their triggers, target models, and Python code where available.

Security mapping

Access rights (ir.model.access) and record rules (ir.rule) tied to custom models.

Migration planning

A Markdown report with executive summary, grouped tables, and a mapping to the equivalent Odoo module layout.
